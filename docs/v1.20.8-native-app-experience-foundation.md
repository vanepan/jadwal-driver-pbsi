## 1 · Native Experience Audit

State of the app before this sprint, established via a 10-area codebase survey.

| Area | Finding |
|---|---|
| Browser feel | No `touch-action` anywhere; tap-highlight and `user-select` handled ad hoc in ~20 scattered rules; no `overscroll-behavior-y`. |
| Startup | `js/app.js` (10,948 lines) statically imported ~90 modules with zero role-gating — a driver session downloaded the same JS as an admin. |
| Drawer | Button/overlay/Escape only — no swipe gesture, despite the codebase already having one swipe idiom to copy (`request-mode-selector.js`). |
| Bottom nav | 4 fixed HTML buttons, visibility toggled per role — could not express 4–5 role-specific tabs with different destinations. |
| Notifications | Deep-link plumbing was fully built and completely dead — the service worker fired a `pbsi:push-nav` event that zero code listened for. Read state was one global cutoff timestamp; no per-item read/archive. |
| Loading states | Home workspace already had a proper animated skeleton (missed by the initial static-file audit — it's injected via JS, not a `.css` file). Analytics KPI skeleton existed but never animated. |
| Offline | Zero `navigator.onLine` handling anywhere — no in-session indicator when connectivity dropped mid-use. |

## 2–3 · Startup & Lazy-Loading Report — the 10 Phases

Executed sequentially; each phase gated on a syntax check, a regression run, and an architecture-consistency check before the next began.

### Phase 1 — Eliminate Browser Feeling `Shipped`
*Objectives 1, 8*

Global `touch-action: manipulation`, app-wide tap-highlight/text-selection kill on every real button, `overscroll-behavior-y: contain`. Bumped two under-48dp controls (`.sidebar-nav-item`, `.btn-icon`) to touch-target minimums.

Files: `style.css` · `index.html`

### Phase 2 — Motion System Foundation `Shipped`
*Objective 10 (tokens)*

Six `--motion-*`/`--ease-*` custom properties plus one global `prefers-reduced-motion` guard, covering every animation added this sprint and every animation that already existed.

Files: `style.css`

### Phase 3 — PWA Polish `Shipped`
*Objective 13*

`viewport-fit=cover` plus safe-area insets on all three top-edge bars; splash background aligned to the manifest's `background_color` so Android's OS-drawn splash doesn't flash a mismatched tone. Found and fixed a pre-existing CSS parse error as a drive-by.

Files: `index.html` · `style.css` · `platform.css`

### Phase 4 — Notification Deep-Link + Center Redesign `Shipped`
*Objectives 6, 7*

Wired the dead `pbsi:push-nav` event to the assignment/request/engineering opener it should have called all along. Rebuilt the Notification Center into Unread/Today/Yesterday/Earlier, with genuine per-item Mark Read and Archive backed by a new additive RTDB node that mirrors the existing `push_subscriptions` pattern — `/notifications` itself stays untouched and server-authoritative.

Files: `js/notifications.js` · `js/app.js` · `js/requests.js` · `js/engineering/ui/engineering-center.js` · `database.rules.json`

### Phase 5 — Native Drawer Swipe `Shipped`
*Objective 4*

Edge-swipe-to-open, swipe-to-close, live-following the drag with the existing CSS transition — copied the touch gesture shape already proven elsewhere in the codebase rather than inventing a new one. Desktop mouse interaction is untouched.

Files: `js/app.js`

### Phase 6 — Role-Based Bottom Navigation `Shipped`
*Objective 5 — largest phase*

4 fixed buttons became a data-driven registry keyed by workspace id — the same join key the Workspace Registry already uses. Shipped three new screens the role-specific tab list needed: a dedicated Driver History screen, an Engineering "Pekerjaan" personal work-queue, and an admin "Lainnya" bottom sheet (a new reusable primitive). A recurring "is this assignment mine" predicate, independently duplicated three times, was consolidated into one shared helper along the way.

Files: `js/config/bottom-nav-registry.js` **(new)** · `index.html` · `js/app.js` · `js/driver-dashboard.js` · `js/engineering/ui/engineering-atoms.js` · `engineering-center.js` · `engineering-queue.js` · `engineering-views.js` · `style.css`

### Phase 7 — Skeleton Loading `Shipped`
*Objective 9*

Added reusable `.skel-*` utilities on a shared pulse keyframe, then used them to fix the one real gap found: the Analytics KPI skeleton was a *static* gray block with no animation. Deliberately did not bolt skeletons onto the new Driver History / My Jobs screens — both render synchronously from already-resident data, so there is no loading gap to bridge there.

Files: `style.css` · `platform.css`

### Phase 8 — Offline Experience `Shipped`
*Objective 11*

Debounced `online`/`offline` listeners routed through the existing toast primitive — no new banner component, no workflow interruption.

Files: `js/app.js`

### Phase 9 — State Restoration `Shipped (narrowed)`
*Objective 12*

Persists and restores the active module, search query, and scroll position by calling the exact same navigation functions a click would. Deliberately did not restore into a specific open assignment/report/modal — that state lives independently inside each module (driver dashboard view, engineering's own screen state, analytics section) and restoring it safely would mean touching every module's internals, not a bolt-on addition.

Files: `js/app.js`

### Phase 10 — Startup Optimization + Lazy Loading `Scoped`
*Objectives 2, 3 — 1 of 5 planned steps executed*

Converted the exports/analytics/* bundle (6 files, 1,875 lines) to dynamic `import()` using the exact loader-map-plus-cache idiom the Workspace widget registry already established — loaded once, on first real export attempt. The remaining 4 staged conversions (vehicle-prediction/simulation-panel, the large analytics engine cluster, engineering-analytics, petty-cash-analytics) are fully designed in the working plan but were not mechanically executed — see §9.

Files: `js/exports/export-registry.js` · `js/app.js`

## 5–6 · Performance & Mobile UX Improvements

- **1,875 fewer lines parsed at boot** for every session — the export bundle now loads only when someone actually exports something.
- **No more tap delay or accidental text selection** on any real button, app-wide, in one global rule instead of ~20 scattered ones.
- **Notifications now actually deep-link** — tapping a push notification opens the assignment/request/engineering item directly instead of landing on whatever page happened to be open.
- **Bottom nav matches the job** — Driver, Engineering, Bidang, and Admin each see their own 4–5 tabs instead of a shared 4-slot set with visibility toggles.
- **The app remembers where you were** — module, search, and scroll position survive a reopen.
- **Content no longer sits under the notch/status bar** on devices with a safe-area inset, now that the viewport intentionally extends edge-to-edge.

---

## 7 · Accessibility Improvements

- One global `prefers-reduced-motion` guard now covers every animation in the app — new and pre-existing — collapsing durations to near-zero for users who've asked for it.
- Two previously sub-44px controls (`.sidebar-nav-item`, `.btn-icon`) raised to the 44–48dp touch-target minimum.
- Pinch-zoom was deliberately *not* disabled — double-tap-zoom and tap delay are killed via `touch-action: manipulation` instead of the accessibility-hostile `user-scalable=no`.
